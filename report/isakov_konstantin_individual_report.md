## 1. Моя роль в проекте

**Роль**: Backend Developer

**Основные обязанности:**

* Разработка backend-функциональности модуля транзакций (Transactions)
* Проектирование и реализация API-эндпоинтов
* Работа с моделями данных и миграциями БД
* Реализация фильтрации, сортировки и пагинации

## 2. Перечень выполненных задач

#### Feature: Сделать каркас API для транзакций (#8)

* Доработана SQLAlchemy-модель Transactions
* Реализованы pydantic схемы для транзакций
* Настроены связи с User и Group
* Реализованы эндпоинты по CRUD
* Реализована фильтрация и пагинация для транзакций
* Разработана документация для тестирования транзакций

**Результат:** Модель транзакций корректно создана и готова к использованию в API.

---

#### Feature: Доработать API для транзакций, добавить привязку к одной или нескольким группам (#17)

* Редактированы эндпоинты для транзакций
* Редактирована связь с группами пользователей
* Реализована возможность добавления транзакций и ее отображение в нескольких группах
* Настроены значения по умолчанию
* Добавлена сортировка транзакций по дате добавления
* Реализован эндпоинт статистики по группе

**Результат:** Модель транзакций работает корректно с одной либо несколькими группами.

---

#### Feature: Добавить напоминания о регулярных платежах (#25)

* РедактиованаSQLAlchemy-модель Transactions для возможности добаления регулярных транзакций
* Редактированы pydantic модели для корректной работы с регулярными транзакциями
* Реализован AsyncIOScheduler - планировщик для выполнения регулярных транзакций, а также для вывода в консоль напоминаний о регулярных платежах дважды в сутки
* Реализован эндпоинт для принудительного вызова напоминаний call_reminders для упрощения тестирования функционала
* Добавлен эндпоинт get_recurring_transactions для просмотра регулярных транзакций
* Добавлен эндпоинт upcoming_reminders для просмотра предстоящих платежей на заданное количество дней

**Результат:** Пользователь может получать напоминания о регулярных платежах.

## 4. Саморефлексия

При выполнении проектного практикума я впервые попробовал работу в команде. Я не имею опыта практической разработки, но, по моему мнению, работа было организована в условиях, приближенных к реальным: с использованием GitHub канбан-доски и распределением декомпозированных задач между участниками. По мере выполения проекта я впервые столкнулся с FastAPI, увидел backend структуру реального приложения, способы ее реализации, взаимодействие с базой данных.

**Моя работа в проекте** в основном была связана с разработкой функционала работы с транзакциями — я реализовал CRUD эндпоинты для транзакций, организовал бизнес логику для взаимодействия пользователей и групп с транзакциями. Кроме этого, мной была реализована фильтрация и пагинация для транзакций. Заключительным этапом моей работы была реализация напоминаний о регулярных платежах, где я познакомился с работой планировщика, через который реализовал необходимый функционал. 

Наиболее **сложным этапом** разработки для меня стал начальный этап, выполнение первых задач, так как не имея необходымых знаний первый вариант реализации для транзакций был выполнен путем переписывания "по аналогии" с моделей пользователя и групп. Понимание и структурное знание пришло после неоднократных исправлений ошибок и нерабочего кода. Только после этого, в рамках выполнения второй своей задачи, была проведена доработка эндпоинтов, сделанных ранее.

От **работы в команде** у меня остались положительные впечатления. Каждый член команды внес свой вклад в реализацию проекта и выполнения задания. Взаимодействие было организовано на высоком уровне, в команде велись обсуждения как общей структуры проекта, так и каких-то конкретных моментов реализации определенного функционала. От всех членов команды ощущалась заинтересованность в общем деле, присутствовала оперативная обратная связь при внесении корректировок от тимлида. Кроме этого, каждый был готов помочь при возникновении трудностей в процессе выполнения задач.
