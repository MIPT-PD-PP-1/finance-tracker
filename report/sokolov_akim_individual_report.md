## 1. Моя роль в проекте

**Роль**: Backend Developer

**Основные обязанности:**

* Разработка части backend-функциональности модулей пользователей и групп пользователей (users, groups).
* Проектирование и реализация API-эндпоинтов для просмотра финансовой статистики пользователя и группы пользователей.
* Работа с pydantic схемами.
* Реализация фильтрации, группировки.

## 2. Перечень выполненных задач

#### Feature: Сделать каркас API для статистики (#18)

* Реализованы эндпоинты для просмотра финансовой статистики пользователя и группы пользователей.
* Реализована фильтрация для получаемой статистки.
* Реализована группировка расходов по категориям и временным периодам для получаемой статистики.

**Результат:** Пользователь может получать персональную аналитику и аналитику для группы, в которой он состоит.

## 4. Саморефлексия

При выполнении проектного практикума я выполнял роль бэкенд разработчика, что стало для меня первым опытом разработки бэкенда приложения, в том числе первым опытом разработки в команде.

**Моя работа в проекте** была связана с разработкой части функционала в модулях users и groups - эндпоинтов для получения финансовой статистики пользователя и группы пользователей. Я реализовал логику для получения статистики, включая возможность применения различных фильтров и группировку расходов по категориям и временным периодам. 

Наиболее **сложным этапом** разработки для меня были этапы понимания общей стуктуры проекта, кода, разработанного коллегами по команде, изучения взаимодействия с базой данных.
